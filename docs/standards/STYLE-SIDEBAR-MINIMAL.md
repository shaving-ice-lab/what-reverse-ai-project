## 1. 设计目标

- **文字对齐优先**: 展开态导航不放左侧图标，确保文字垂直对齐
- **紧凑窄栏**: 降低侧边栏宽度，提升内容区可用面积
- **低干扰呈现**: 统一文本按钮与轻量 hover/active，避免装饰性视觉

## 2. 视觉基调

- **风格关键词**: Minimal / Quiet / Text-first / Compact
- **标识性元素**: 统一行高与低对比底色，强调秩序与简洁
- **层次策略**: 依靠轻微背景与文字色变化区分状态

## 3. 字体与排版

- **主字体**: 使用系统默认无衬线字体
- **字号**: 导航项 `12px`，次级信息 `11px`
- **字重**: 导航项 `500`，次级信息 `400`
- **对齐**: 文字左对齐，不使用前缀符号

## 4. 颜色与对比

- **默认文字**: `text-foreground-light`
- **Hover 状态**: `text-foreground` + `bg-surface-100/60`
- **Active 状态**: `text-foreground` + `bg-surface-100/70`
- **边框**: 仅用于侧边栏边界与分隔线

## 5. 背景与纹理

- **背景策略**: 纯净背景，不使用扫描线/噪点纹理
- **层次方式**: 通过轻微底色与边框形成层次

## 6. 组件语言

- **导航项**: 纯文本行，无左侧图标，统一行高
- **折叠态**: 可显示图标与 Tooltip 提示
- **新建入口**: 与普通导航保持一致，不使用强按钮样式
- **底部折叠**: 同样使用文本行样式保持一致性

## 7. 交互与动效

- **动效原则**: 低幅度、短时长、仅颜色过渡
- **建议动画**: `transition-colors`，150-200ms
- **避免**: 缩放、发光、厚重阴影

## 8. 布局与空间

- **宽度**: 展开 `188px`，折叠 `52px`
- **内边距**: 展开 `8px` (`px-2`)，折叠 `6px` (`px-1.5`)
- **行高**: `32px` (`h-8`)
- **分组策略**: 文本列表连续，减少多余分隔

## 9. 可访问性与性能

- **对比度**: 主文本与背景至少 4.5:1
- **可点区域**: 行高 `32px` 以上确保可点击
- **性能**: 避免复杂纹理与高强度阴影

## 10. 实现要点与变量

### 关键文件

- `apps/web/src/app/(dashboard)/layout.tsx`

### 关键参数

- 侧边栏宽度: `w-[188px]` / 折叠 `w-[52px]`
- 行高: `h-8`
- 默认文本: `text-foreground-light`
- Hover: `hover:bg-surface-100/60`
- Active: `bg-surface-100/70 text-foreground`

### 推荐变量

```
--sidebar-width: 188px;
--sidebar-width-collapsed: 52px;
--sidebar-item-height: 32px;
--sidebar-padding: 8px;
--sidebar-padding-collapsed: 6px;
```
